Public API (beta)

The Rock public API allows you to integrate your own applications with Rock and programmatically create tasks, send messages, and perform other actions. The API so far only includes the minimal set of methods to create new objects in Rock, there are no methods for

listing existing objects or updating/deleting those.

The public API works on behalf of a “space bot”, a special space member that inherits the permissions from the one who created it in a space - the messages/tasks/etc created in the spaces are created on behalf of that special space member. This also means that a public API instance is scoped to a specific space, there are no APIs to do any requests cross-space.

Note that the public API is still in beta stage and may be subject to changes.

Enabling the public API

The public API can be enabled in a specific space by creating a “Custom Webhook” bot in that space, getting an authentication token for that bot and doing API requests using that token (naturally, to access multiple spaces a bot has to be created in each space and different tokens are to be used to access each space). To create a “Custom Webhook” bot in a space (same as for any other bot):

  1. The space has to be made PRO by some user (who has a paid plan)
  2. The one creating a bot has to be the space admin

With the above in place one can click the "gear" icon at the top right of the space view (in the web app), find "Integrations" there, click "+" and select "Custom Webhook", then select "Permissions" to enable for this webhook instance, and click "Create". This would create a space bot and show the corresponding authentication token to trigger the webhook. The authentication token should be copied/stored by the user - as it's only exposed once in that dialog box, there’s no way to retrieve it later after the dialog box is dismissed (the user has to remove the existing bot and create a new one with a new token).

The “Custom webhook” bot would now be listed in “Integrations”, and otherwise appear in the space as a normal space member. Notes/tasks/messages created/sent via the public API using the token associated with it will have this space bot as their author/creator. The bot possesses the same permissions its creator has in that space, with extra restrictions applied via “Permissions” checkboxes list in the “Create” dialog. It is not possible to use the bot to impersonate any other member of the space (including other bots).

Sending requests

The public API is accessed by sending HTTPS (HTTP/1.1 or HTTP/2, TLS1.2+) requests to the following base URL: https://api.rock.so/webhook/bot

To authenticate, you can use any of the two authentication methods blow:

  1. Append the auth query parameter with the value being the authentication token.
  2. Use the Authorization HTTP header with the value set to Bearer <token>.

The public API provides several methods for retrieving and manipulating data. The choice of a method is done by adding the method query parameter to the request.

General conventions

The Create and Send methods require HTTP POST to be used, while the Get and List methods require GET. The Create and Send methods expect JSON-encoded data as input. Most fields are optional, unless denoted otherwise.

On success, the API returns either 200 or 204 status code; in the former case, the response body will contain a JSON-encoded message. In case of a 400 HTTP response the response body, if not empty, contains a JSON response describing the error. The schema for JSON requests and responses is provided below for each method individually.

Other HTTP status codes are to be interpreted according to their standard protocol meaning.

Pagination

The ListXXX methods provide a unified way of pagination by accepting cursor and limit query parameters. The cursor is a pagination cursor: it has to be passed empty on the first invocation. Then, if the response hasMore property is true, the nextCursor from the response has to be passed as cursor to the next call. If hasMore is false / missing, the iteration is completed.

Available methods

createNote

The createNote method allows you to create notes.

Request

CreateNoteRequest {
        body []TextFragment
        labels []string
        watchers []UserID
}

The body must be present and cannot be empty.

Response

CreateNoteResponse {
        id uint32
}

CreateNoteErrorResponse {
        error string
}

On success, the ID of the newly-created note is returned.

Example

Request

POST https://api.rock.so/webhook/bot?auth=xxx&method=createNote

{
        "body": [
                {"text":"Note this down"}
        ]
}

Response

{"id":1}

createTask

The createTask method allows you to create a new task.

Request

CreateTaskRequest {
        title string
        body []TextFragment
        listId uint32
        start uint32
        due uint32
        owners []string
        priority int32
        severity int32
        sprint uint32
        checkList []string
        recurringSchedule RecurringSchedule
        customFields map[string]string
        labels []string
        watchers []string
}

There are several constraints imposed on the input:

  • listID must be present and have a non-zero value.
  • priority must have a value between -2 (lowest) and +2 (highest). If omitted, the default value is 0 (medium).
  • title must be present and have a length between 1 and 1024 characters.

Response

CreateTaskResponse {
        id uint32
}

CreateTaskErrorResponse {
        error string
}

On success, the ID of the newly-created task is returned.

Example

Request

POST https://api.rock.so/webhook/bot?auth=xxx&method=createTask

{
        "title":"Water the plants",
        "listId": 1,
        "labels": ["home"]

}

Response

{"id":3}

listLabels

The listLabels method can be used to retrieve the list of labels assigned to tasks. Note that labels are created and deleted "on the fly"; when there are no more tasks with a given label (either the label was removed from all tasks, or all tasks with the label were deleted), that label is deleted.

Response

ListLabelsResponse {
        labels []string
        hasMore bool
        nextCursor string
}

Example

Request

GET https://api.rock.so/webhook/bot?auth=xxx&method=listLabels

Response

{
 "labels": [
   "Education", "Finances", "Home"
 ],
 "hasMore": true,
 "nextCursor": "Zml2ZQ"
}

sendMessage

The sendMessage method allows you to send a new message.

Request

SendMessageRequest {
        text string
        parentMsgId uint32
}

The message text uses similar formatting rules as when sending messages via the app.

  • @At-signs can be used for referencing people, spaces, and objects:
  • To mention a space member, use the @user:ID$ notation.
  • To refer to another space, use the @space:ID$ notation.
  • To refer to a space object, use the @TYPE:ID$ notation, where TYPE can be one of: attachment, note, task or topic.
  • Objects in another space can be referenced by combining the space reference and object reference using a @space:ID/TYPE:ID$ notation.
  • *Asterisks* can be used for bold text.
  • _Underscores_ can be used for italic text.
  • ~Tildes~ can be used for striked-out text.
  • `Backticks` can be used for monospace text.
  • ```Triple backticks``` can be used for code blocks.

The parentMsgId field can be used to submit a message as a response to another message. If the parent message is part of an existing thread, the new message will also appear in said thread.

Response

SendMessageResponse {
        id uint32
}

On success, the ID of the newly-created message is returned.

Example

Request

POST https://api.rock.so/webhook/bot?auth=xxx&method=sendMessage

{
        "text": "Hello! This is an *automated message*."

}

Response

{"id":24}

Examples

When using these examples, substitute XXYYZZ for your authentication token.

curl

List labels

curl 'https://api.rock.so/webhook/bot?auth=XXYYZZ&method=listLabels'

Send message

curl
 --data '{"text":"Hello! This is an *automated message.*"}'
 'https://api.rock.so/webhook/bot?auth=XXYYZZ&method=sendMessage'

Google Apps Script

Get bot info

var response = UrlFetchApp.fetch(
 'https://api.rock.so/webhook/bot?auth=XXYYZZ&method=getBotInfo'
);
var data = JSON.parse(response.getContentText());

Create task

var data = {

  'title': 'Buy groceries',

  'listId': 1,

  'due': 1672531200,

};

var options = {

  'method' : 'post',

  'contentType': 'application/json',

  'payload' : JSON.stringify(data)

};

UrlFetchApp.fetch(
 'https://api.rock.so/webhook/bot?auth=XXYYZZ&method=createTask',
 options
);
